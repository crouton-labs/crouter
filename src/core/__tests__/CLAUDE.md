- Never pass `{ kind: 'stdin', ... }` params to `parseArgv` without a positional token to satisfy them — `parseArgv` reads stdin to EOF for any unsatisfied `stdin` param, and `node --test` pipes stdin with no EOF signal, so the test hangs forever with no output or error. Either provide a positional token (see `argv-parser.test.ts` `stdin-as-positional` suite) or omit the `stdin` param.
