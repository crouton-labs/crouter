- Leaf handlers receive **camelCase** keys for flags, not kebab-case: `--context-file` → `input['contextFile']`, `--my-flag` → `input['myFlag']`. Accessing the kebab name returns `undefined` silently. (`parseArgv` in command.ts applies `flagNameToKey` before calling `run`.) Positional and stdin params arrive under their declared name as-is (no transform).
- The user-facing flag for any `kind: 'context-file'` param is **always literally `--context-file`** — the param's `name` field only routes the parsed value into `input[name]`. Declaring a context-file param with `name: 'deck'` does NOT create a `--deck-file` flag; callers still pass `--context-file deck.json` and the handler reads `input['deck']`. (Hardcoded in `parseArgv` and `renderLeafArgv` in `command.ts` / `help.ts`.)
- `outputKind: 'jsonl'` in `LeafHelp` is a display hint only — the framework does not enforce it. JSONL leaves must call `emitLine()` themselves and return `void`; calling `emit()` produces multi-line pretty JSON that silently breaks downstream line parsers.
